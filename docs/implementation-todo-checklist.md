## 阶段 1：采集 MVP（Day 1 ~ Day 4）

### TODO 1.1 事件模型定义（最关键）
- [ ] 定义 `RawEvent`（原始事件）。
- [ ] 定义 `ContextSnapshot`（上下文快照：app/window）。
- [ ] 定义 `NormalizedEvent`（标准化事件）。

**输出物**
- `core/capture` 下 schema 文档或类型定义。

**验收标准**
- [ ] 能覆盖“鼠标点击 + 前台应用 + 时间戳 + 会话 ID”。

### TODO 1.2 采集引擎最小实现
- [ ] macOS 权限检查（辅助功能权限）。
- [ ] 监听鼠标点击坐标。
- [ ] 获取当前前台应用和窗口标题。
- [ ] 将事件写入本地队列。

**输出物**
- 可运行的采集进程（CLI 或后台服务）。

**验收标准**
- [ ] 连续点击 20 次，事件不丢失。
- [ ] 权限不足时给出明确提示。

### TODO 1.3 事件落盘与轮转
- [ ] JSONL 落盘实现。
- [ ] 以 session 分文件（例如按日期+sessionId）。
- [ ] 日志轮转策略（大小/时间）。

**输出物**
- `data/raw-events/*.jsonl`

**验收标准**
- [ ] 文件可被 `jq` 正确解析。
- [ ] 异常中断后数据文件可继续追加。

## 阶段 2：知识建模 MVP（Day 4 ~ Day 7）

### TODO 2.1 任务切片器（Session -> Task）
- [ ] 按空闲间隔/窗口切换识别任务边界。
- [ ] 给每段任务生成 `task_id`。

**输出物**
- `TaskChunk` 结构与切片规则。

**验收标准**
- [ ] 可将 1 个 session 自动切分为多个任务片段。

### TODO 2.2 知识条目格式定义
- [ ] 定义 `KnowledgeItem`（目标、步骤、上下文、约束）。
- [ ] 增加版本号字段（schemaVersion）。

**输出物**
- `data/knowledge/*.json`

**验收标准**
- [ ] 任意任务切片都能映射为合法 `KnowledgeItem`。

### TODO 2.3 自动总结初版（无 LLM）
- [ ] 用规则生成步骤摘要（例如“打开 X -> 点击 Y -> 输入 Z”）。
- [ ] 输出老师可读的摘要文本。

**输出物**
- `summary` 字段及其生成模块。

**验收标准**
- [ ] 10 条样例任务中，摘要可读性通过人工检查。

### TODO 3.1 提示词模板系统

**验收标准**

**输出物**

**验收标准**

- [ ] 增加字段校验与 fallback。

**输出物**

**验收标准**
- [ ] 至少 3 个示例任务成功转换并可被 OpenClaw 读取。
